# План тестирования проекта

## 1. Цель тестирования

Цель тестирования — проверить корректность работы базы данных, ограничений целостности и клиентского Java-приложения при работе с двумя СУБД: PostgreSQL и MySQL.

## 2. Проверка базы данных

| № | Проверка | Действие | Ожидаемый результат |
|---|----------|----------|---------------------|
| 1 | Создание таблиц | Запустить `01_create_schema.sql` | Все таблицы созданы без ошибок |
| 2 | Заполнение данными | Запустить `02_insert_test_data.sql` | В каждой таблице есть не менее 5 записей или связей |
| 3 | Первичные ключи | Попытаться добавить дубликат в таблицу-связку | Ошибка нарушения PRIMARY KEY |
| 4 | Внешние ключи | Добавить рецепт с несуществующим `author_id` | Ошибка FOREIGN KEY |
| 5 | Проверка рейтинга | Добавить отзыв с оценкой 10 | Ошибка CHECK |
| 6 | Проверка времени | Добавить рецепт с временем -10 | Ошибка CHECK |
| 7 | Уникальность | Добавить пользователя с уже существующим email | Ошибка UNIQUE |

## 3. Проверка приложения

| № | Проверка | Действие | Ожидаемый результат |
|---|----------|----------|---------------------|
| 1 | Подключение к PostgreSQL | Запустить приложение с `app-postgres.properties` | Подключение выполнено |
| 2 | Подключение к MySQL | Запустить приложение с `app-mysql.properties` | Подключение выполнено |
| 3 | Просмотр рецептов | Выбрать пункт 1 | Выводится список рецептов |
| 4 | Просмотр карточки | Выбрать пункт 2 и ввести id | Выводятся ингредиенты и шаги |
| 5 | Поиск | Выбрать пункт 3 и ввести часть названия | Выводятся подходящие рецепты |
| 6 | Добавление рецепта | Выбрать пункт 4 и заполнить поля | Новый рецепт появляется в списке |
| 7 | Добавление отзыва | Выбрать пункт 5 | Отзыв добавляется, рейтинг меняется |
| 8 | Обработка ошибок | Ввести неверный id или неверную оценку | Приложение выводит ошибку БД и не завершается аварийно |

## 4. Проверка на двух ОС

Проверить проект на двух операционных системах:

1. Основная ОС, например Windows 10/11.
2. Вторая ОС, например Ubuntu в виртуальной машине.

Для каждой ОС нужно проверить:

- установлен ли JDK 17;
- установлен ли Maven;
- запускается ли приложение;
- корректно ли читается файл настроек;
- работает ли подключение к выбранной СУБД;
- корректно ли отображается русский текст.

## 5. Итог тестирования

Проект считается успешно протестированным, если:

- SQL-скрипты выполняются без ошибок;
- тестовые ограничения отклоняют неправильные данные;
- приложение запускается с PostgreSQL и MySQL;
- основные функции приложения работают;
- другой студент смог развернуть проект по README без личной помощи автора.
